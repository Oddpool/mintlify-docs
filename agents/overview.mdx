## The fastest way: use the onboarding skill

If you use a coding agent like [Claude Code](https://claude.ai/claude-code), [Codex](https://github.com/openai/codex), or [OpenCode](https://github.com/opencode-ai/opencode), install our onboarding skill and it will do everything for you.

**Install the skill:**

```bash
curl -fsSL https://docs.benchspan.com/install-skill.sh | sh
```

This installs the `/onboard-agent` skill for your coding agent. Then run it:

```bash
/onboard-agent
```

The skill will:

1. Ask about your agent (where's the code, what runtime, what env vars it needs)
2. Explore your codebase to understand how to invoke it
3. Generate a working `runner.sh`
4. Help you set up env vars on the dashboard
5. Run the [healthcheck benchmark](/agents/healthcheck) to verify everything works
6. Diagnose and fix any failures automatically

Most agents are fully onboarded in a single session.

<Info>
  The skill follows the [Agent Skills](https://agentskills.io) open standard and works with Claude Code, Cursor, Codex, GitHub Copilot, and other compatible agents.
</Info>

## Or do it manually

If you prefer to do it yourself, here's the process:

<Steps>
  <Step title="Understand how it works">
    Your agent runs in a Docker container. It gets a problem statement and a working directory. It produces output. Benchspan grades the result.

    You write one file — `runner.sh` — that installs and runs your agent. That's the entire integration.

    [How it works](/agents/how-it-works)
  </Step>
  <Step title="Write your runner.sh">
    Follow the pattern that matches your setup: pip package, npm package, binary, or build from source.

    [Writing your runner.sh](/agents/writing-runner-sh)
  </Step>
  <Step title="Set env vars on the dashboard">
    Add your agent's required env vars (API keys, model config) at your [dashboard settings](https://benchspan.com/dashboard/settings/profile). Whatever you set there gets injected into the container.
  </Step>
  <Step title="Test with the healthcheck benchmark">
    Run 10 simple tasks that verify your runner.sh works across different environments.

    ```bash
    benchspan run --benchmark agent-healthcheck --agent ./my-agent
    ```

    [Healthcheck details](/agents/healthcheck)
  </Step>
  <Step title="Run a real benchmark">
    ```bash
    benchspan run --benchmark swebench --agent ./my-agent --instances 5
    ```
  </Step>
</Steps>

## What you'll need

- A Benchspan account ([sign up](https://benchspan.com))
- The CLI installed (`pip install benchspan`)
- Your agent's source code or published package
- API keys for whatever LLM your agent uses
