---
title: "Error Reporting"
description: "Surface custom errors from your agent to the dashboard."
---

When something goes wrong during a run, BenchSpan captures the error and shows it in the dashboard and CLI. This happens automatically for container-level failures (crashes, timeouts, non-zero exits).

But sometimes your agent knows *why* it failed — a missing API key, a malformed input, a model refusal. You can report these by writing an `error.json` file to `$OUTPUT_DIR`.

## Writing error.json

Write a JSON file with a `message` field:

```json
{
  "message": "ANTHROPIC_API_KEY returned 401 — check your key on the dashboard"
}
```

That's the minimum. The message shows up in the dashboard's instance list and in `benchspan runs show`.

You can add `source` and `type` for more context:

```json
{
  "source": "agent",
  "type": "AuthenticationError",
  "message": "ANTHROPIC_API_KEY returned 401 — check your key on the dashboard"
}
```

This renders as `[agent] AuthenticationError: ANTHROPIC_API_KEY returned 401` in the CLI and dashboard.

## Field reference

| Field | Type | Required | Description |
|---|---|---|---|
| `message` | string | Yes | Human-readable error description |
| `source` | string | No | Where the error originated: `"agent"`, `"container"`, `"harness"` |
| `type` | string | No | Error class name: `"AuthenticationError"`, `"TimeoutError"`, etc. |

You can add any other fields you want (stack traces, request IDs, debug info). The full `error.json` is preserved in the run artifacts and downloadable via `benchspan runs download`.

## Example in runner.sh

```bash
#!/bin/bash
set -uo pipefail

# Check API key before doing anything expensive
if [ -z "${ANTHROPIC_API_KEY:-}" ]; then
  echo '{"source":"agent","type":"ConfigError","message":"ANTHROPIC_API_KEY not set"}' \
    > "$OUTPUT_DIR/error.json"
  exit 1
fi

# Run your agent
my-agent --task "$PROBLEM_STATEMENT" 2>"$OUTPUT_DIR/stderr.log" || {
  echo '{"source":"agent","message":"Agent crashed — see stderr.log for details"}' \
    > "$OUTPUT_DIR/error.json"
  exit 1
}
```

## What BenchSpan captures automatically

You don't need to write `error.json` for these — they're handled for you:

- **Container exits non-zero** — `[container] Container exited with code 1`
- **Container times out** — `[container] Container timed out after 1800s`
- **Infrastructure errors** — worker failures, image pull errors, etc.

Your `error.json` takes priority. If you write one, it replaces the default container error message.

## Viewing errors

**CLI:**
```bash
benchspan runs show <run_id>
```
The per-instance table shows errors in the rightmost column, truncated to 60 characters.

**Dashboard:**
Click any failed instance to see the full error with source and type.

**Full details:**
```bash
benchspan runs download <run_id> -i <instance_id>
```
Downloads all artifacts including the complete `error.json` with any extra fields you added.
